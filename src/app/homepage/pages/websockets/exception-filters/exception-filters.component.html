Exception Filters

The websockets exceptions layer works exact same as the prime layer. The only difference is that instead of throwing HttpException, you should throw WsException. 

{{ wsException }}
Notice The WsException class is imported from the @nestjs/websockets package.

Nest would handle this exception and emits the exception message with the following data: 

{{ exceptionResponse }}

Exception Filters

The Exception Filters are also really similar, and works exactly in the same way as the primary ones.

{{ 'ws-exception.filter' | extension: wsExceptionFilterT.isJsActive }} 
{{ wsExceptionFilter }}
{{ wsExceptionFilterJs }}
Notice It's impossible to setup the websockets exception filters globally.
